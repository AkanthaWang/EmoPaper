设计原则

把 README 里的核心约束转成可视化卡片，方便直接演示。

系统原则

  • 数据驱动 UI所有论文信息通过 JSON 统一控制，前端只负责渲染与交互。
  • 视图与内容解耦Vue 负责结构化浏览，Docusaurus 负责深度解释和知识沉淀。
  • 渐进式信息展示Timeline 提供概览，Card 展示摘要，Blog 承载分析深度。

多模态规划

  • Image-based Emotion当前主线，围绕数据集、方法和基准形成完整浏览闭环。
  • Video-based Emotion后续可在相同结构下扩展出视频时间线与新型论文视图。
  • 统一扩展接口通过 modality 字段区分分支，为未来数据库或 API 化预留接口。

系统架构

直接把 README 里的结构图做成可读的层级展示。

Docusaurus 博客系统 承载方法分析、数据集构建说明和 Benchmark 解读，面向深度内容表达。
Vue 时间线系统 主浏览层，负责 Dataset / Method / Benchmark 三视图、卡片浏览、年份分组和标签筛选。
JSON 数据层 统一管理论文元数据，作为时间线与博客系统之间的共享事实源。
Vue → 浏览与交互 Docusaurus → 深度内容 JSON → 单一数据源
